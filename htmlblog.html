HTML Cheat Sheet

HTML (HyperText Markup Language) is the most basic building block of the Web. It defines the meaning and structure of web content. "Hypertext" refers to links that connect web pages to one another, either within a single website or between websites.

HTML Syntax

<!DOCTYPE html> at the top level declares the document type and version of HTML
Space and indentation in code is ignored in HTML. It only affects readability.
HTML elements are the building blocks of HTML. Each element begins with a opening tag <tag> and ends with a closing tag <tag/>.
Void elements have self-enclosing tags (no closing tag) and no content. <img src="image.jpg" alt="Description" >
Head section is for code that will not be displayed on browser.
Body section is for code that will be rendered as content on browser.
HTML enables elements to be nested within other elements. 
          
            <div>
              <h1>Title</h1>
              <p>This is a paragraph inside a div.</p>
            </div>
Comments are written like this: <!--This is a comment -->
JavaScript scripts imported is preferably at the end of the body so that HTML is rendered first, otherwise loading scripts would delay HTML content and the web page may be disoriented.
Document Metadata
Element Attributes Usage
<html> global attributes, xmlns Root element of an HTML document. All other elements must be descendants.
<head> global attributes Contains metadata and info not displayed in the browser: title, JS, CSS, favicons.
<title> global attributes Defines the document’s title shown in the browser’s title bar or tab.
<base> global attributes, href, target Specifies the base URL for relative URLs in the document. Only one per document.
<link> global attributes, href, hreflang, rel, disabled, title, type, media, sizes, imagesizes, crossorigin, imagesrcset, integrity, as, blocking, fetchpriority, referrerpolicy Defines relationships with external resources: stylesheets, favicons, icons, etc.
<meta> global attributes, charset, content, http-equiv, media, name Represents metadata that other HTML elements cannot represent: <base>, <link>, <script>, <style>, or <title>.
<style> global attributes, blocking, media, nonce, title Contains CSS for the document or part of it.
<script> global attributes, async, attributionsrc, blocking, crossorigin, defer, fetchpriority, integrity, nomodule, nonce, referrerpolicy, src, type Contains executable code, usually JavaScript. Can also contain WebGL, JSON, etc.

Content Sectioning
Element Attributes Usage
<body> global attributes, event attributes Represents the content of an HTML document. Only one per document.
<header> global attributes Represents a header for its nearest ancestor section or sectioning root element.
<footer> global attributes Represents a footer for its nearest ancestor section or sectioning root element.
<section> global attributes Defines a generic section of a document; should have a heading.
<nav> global attributes Provides navigation links within or between documents.
<aside> global attributes Represents content indirectly related to the main content, like sidebars or call-out boxes.
<div> global attributes A generic container for flow content. Block-level element.

Text Content
Element Attributes Usage
<h1> - <h6> global attributes Defines section headings, from largest to smallest.
<p> global attributes Groups related content like text, images, or form fields.
<blockquote> global attributes, cite Indicates a quotation, often with indentation. Use the `cite` attribute for the source.
<cite> global attributes Marks the title of a cited work.
<ol> global attributes Defines an ordered list of items.
<ul> global attributes Defines an unordered list of items.
<li> global attributes Defines an item in an ordered or unordered list.
<dl> global attributes Defines a description list for terms and their definitions.
<dt> global attributes Defines a term in a description list.
<dd> global attributes Defines the description for the preceding term in a description list.
tablefoot
Inline Text Semantics
Element Attributes Usage
<a> global attributes Anchor element, with its href attribute, creates a hyperlink to web pages, files, email addresses, locations in the same page, or anything else a URL can address.
<strong> global attributes Indicates that its contents have strong importance, seriousness, or urgency. Browsers typically render the contents in bold type.
<em> global attributes Marks text that has stress emphasis. Can be nested, with each level of nesting indicating a greater degree of emphasis.
<mark> global attributes Represents text which is marked or highlighted for reference or notation purposes due to the marked passage's relevance in the enclosing context.
<abbr> global attributes Represents an abbreviation or acronym.
<code> global attributes Displays code snippets.
<pre> global attributes Represents preformatted text which is to be presented exactly as written in the HTML file. By default, it is a block-level element, i.e., its default display value is block.
<span> global attributes Generic inline container for phrasing content, which does not inherently represent anything. Inline-level element.
<br> global attributes Produces a line break in text (carriage-return). It is useful for writing a poem or an address, where the division of lines is significant.

Forms and Input
Element Attributes Usage
<form> global attributes, accept-charset, autocapitalize, autocomplete, name, rel, action, enctype, method, novalidate, target Represents a document section for taking user input using interactive controls.
<input> global attributes, type: {button, checkbox, color, date, datetime-local, email, file, hidden, image, month, number, password, radio, range, reset, search, submit, tel, text, time, url, week} Used to create interactive controls for web-based forms.
<button> global attributes, autofocus, command, commandfor, disabled, form, formaction, formenctype, formmethod, formnovalidate, formtarget, name, popovertarget, popovertargetaction, type, value A button can be pressed and it performs an action when pressed.
<label> global attributes, for Represents a caption for an item in a user interface.
<textarea> global attributes, autocapitalize, autocomplete, autocorrect, autofocus, cols, dirname, disabled, form, maxlength, minlength, name, placeholder, readonly, required, rows, spellcheck, wrap Represents a multi-line plain-text editing control, free-form text.
<select> global attributes, autocomplete, autofocus, disabled, form, multiple, name, required, size Represents a control that provides a menu of options.
<option> global attributes, disabled, label, selected, value Used to define an item contained in a <select>, a <optgroup>, or a <datalist> element.
<fieldset> global attributes, disabled, form, name Used to group several controls as well as labels within a web form.

Media
Element Attributes Usage
<img> global attributes, alt, attribution, src, crossorigin, decoding, elementtiming, fetchpriority, height, ismap, loading, referrerpolicy, sizes, srcset, width, usemap Embeds an image into the document.
<audio> global attributes, autoplay, controls, controlslist, crossorigin, disableremoteplayback, loop, muted, preload, src Used to embed sound content in documents.
<video> global attributes, autoplay, controls, controlslist, crossorigin, disablepictureinpicture, disableremoteplayback, height, loop, muted, playsinline, poster, preload, src, width Embeds a media player which supports video playback into the document.
<source> global attributes, type, src, srcset, sizes, media, height, width Specifies one or more media resources for the <picture>, <audio>, and <video> elements. It is a void element, which means that it has no content and does not require a closing tag.
<picture> global attributes Contains zero or more <source> elements and one <img> element to offer alternative versions of an image for different display/device scenarios.
<track> global attributes, default, kind, label, src, srclang Used as a child of the media elements, <audio> and <video>. Each <track> element lets you specify a timed text track (or time-based data) that can be displayed in parallel with the media element, for example to overlay subtitles or closed captions on top of a video or alongside audio tracks.
<iframe> global attributes, allow, allowfullscreen, allowpaymentrequest, browsingtopics, credentialless, csp, height, loading, name, referrerpolicy, sandbox, src, srcdoc, width Represents a nested browsing context, embedding another HTML page into the current one. Consumes significant memory and other computing resources — check for performance issues.
<canvas> global attributes, height, width Use with either the canvas scripting API or the WebGL API to draw graphics and animations.
tablefoot
Tables
Element Attributes Usage
<table> global attributes Represents 2D tabular data: rows × columns.
<thead> global attributes Encapsulates a set of table rows (<tr> elements), indicating that they comprise the head of a table with information about the table's columns.
<tbody> global attributes Encapsulates a set of table rows (<tr> elements), indicating that they comprise the body of a table's (main) data.
<tfoot> global attributes Encapsulates a set of table rows (<tr> elements), indicating that they comprise the foot of a table with information about the table's columns.
<tr> global attributes Defines a row of cells in a table. The row's cells can then be established using a mix of <td> (data cell) and <th> (header cell) elements.
<th> global attributes, abbr, colspan, headers, rowspan, scope Defines a cell as the header of a group of table cells and may be used as a child of the <tr> element.
<td> global attributes, colspan, headers, rowspan Defines a cell of a table that contains data and may be used as a child of the <tr> element.
<caption> global attributes Specifies the caption (or title) of a table, providing the table an accessible description.

Global Attributes
Attributes Usage
accesskey Hint for keyboard shortcuts; space-separated list of characters.
anchor Anchors a positioned element to another using its ID.
autocapitalize Controls automatic capitalization of input text.
autocorrect Enables or disables automatic spelling corrections.
autofocus Automatically focuses the element on page load (boolean).
class Space-separated list of classes for CSS/JavaScript targeting.
contenteditable Specifies if the element is editable by the user.
data-* Custom data attributes for passing proprietary data between HTML and DOM.
dir Specifies text direction: ltr, rtl, or auto.
draggable Indicates whether the element can be dragged (true/false) using the Drag and Drop API.
enterkeyhint Hints what action label (or icon) to present for the enter key on virtual keyboards.
exportparts Exports shadow parts from a nested shadow tree into a containing light tree.
hidden Indicates the element is not relevant or hidden from the user (e.g., during login).
id Defines a unique identifier, which must be unique within the document.
inert Prevents user input events from interacting with the element (boolean).
inputmode Specifies the virtual keyboard configuration for editable content (e.g., elements).
is Specifies that an element should behave like a registered custom-built element.
itemid Unique, global identifier for an item in the HTML Microdata feature.
itemprop Used to add properties to an item in HTML Microdata.
itemref Associates properties with items that are not direct descendants of an element with the itemscope attribute.
itemscope Specifies the scope of an item in HTML Microdata and the vocabulary used.
itemtype Defines the vocabulary URL used to describe item properties in HTML Microdata.
lang Defines the language for non-editable elements or for editable content (BCP 47 format).
nonce Cryptographic nonce used by Content Security Policy (CSP) to determine fetch permissions.
part Specifies the part names of an element for use with shadow DOM styling.
popover Designates an element as a popover (invisible until triggered by another element or API call).
role Defines the semantic meaning of content for assistive technologies like screen readers (ARIA role).
slot Assigns an element to a slot in a shadow DOM.
spellcheck Defines whether an element's text should be checked for spelling errors (true/false).
style Contains CSS styling declarations for the element.
tabindex Defines the element’s focusability and keyboard navigation order.
title Contains advisory information related to the element, often displayed as a tooltip.
translate Specifies if an element’s text should be translated when the page is localized (yes/no).
virtualkeyboardpolicy Controls virtual keyboard behavior for editable content on devices without hardware keyboards (auto/manual).
writingsuggestions Controls the browser's writing suggestions in input fields (true/false).
Global attributes apply to all HTML elements, though they may not affect all.
Additional attributes
XML attributes: xml:lang (specifies the language of the element's content), xml:base (defines a base URI for relative URLs). ARIA roles: role (defines an element's role in the accessibility tree). ARIA attributes: aria-label, aria-labelledby, aria-hidden, aria-expanded, aria-controls, aria-live, aria-disabled, aria-selected, aria-checked, aria-required, aria-invalid.
Event handler attributes:
Form events: onblur, onchange, onfocus, oninput, oninvalid, onreset, onsubmit Mouse events: onclick, ondblclick, onmousedown, onmouseenter, onmouseleave, onmousemove, onmouseover, onmouseout, onmouseup, onmousewheel Keyboard events: onkeydown, onkeypress, onkeyup Media events: onabort, oncanplay, oncanplaythrough, ondurationchange, onemptied, onended, onerror, onloadeddata, onloadedmetadata, onloadstart, onpause, onplay, onplaying, onprogress, onratechange, onseeked, onseeking, onstalled, onsuspend, ontimeupdate, onvolumechange, onwaiting Drag and drop events: ondrag, ondragend, ondragenter, ondragleave, ondragover, ondragstart, ondrop Window events: onresize, onscroll, onload, onunload, onshow, ontoggle
